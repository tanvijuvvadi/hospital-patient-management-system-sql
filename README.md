# 🏥 Hospital Patient Management System (SQL Project)

## 🎯 Objective

The objective of this project is to design and develop a **Hospital Patient Management System** using SQL to efficiently manage hospital operations such as:

- Patient records management
- Doctor and department tracking
- Appointment scheduling
- Treatment and diagnosis history
- Billing and payment tracking
- Hospital analytics and insights

This project simulates a real-world healthcare database system used in hospitals and clinics for operational and analytical decision-making.


## 🛠️ Tools Used

- SQL (MySQL / PostgreSQL compatible)
- Excel (for dataset creation)
- Database Design Concepts
- Relational Schema Modeling

  
## 🧱 Database Schema

The project consists of the following relational tables:

- Patients
- Doctors
- Departments
- Appointments
- Treatments
- Billing
- Admissions


## 📊 Tables Description

### 1. Patients
Stores patient personal information such as name, age, gender, and city.

### 2. Doctors
Stores doctor details including specialization and department mapping.

### 3. Departments
Contains hospital department information like Cardiology, Neurology, etc.

### 4. Appointments
Tracks patient appointments with doctors including date and status.

### 5. Treatments
Stores diagnosis, treatment type, and treatment cost.

### 6. Billing
Tracks billing amount and payment status (Paid / Pending).

### 7. Admissions
Tracks hospital admission and discharge details of patients.

## 🔍 Key SQL Queries

- Fetch all patient records
- Doctor-wise appointment summary
- Department-wise patient count
- Appointment status tracking
- Patient treatment history
- Billing and payment status analysis
- Admission vs discharge tracking


## 📈 Healthcare Insights Generated

- Most visited hospital departments
- Doctors with highest patient load
- Patients with pending bills
- Monthly appointment trends
- Average treatment cost analysis
- Department-wise revenue generation
- Repeat patient analysis
- Emergency case tracking

## 🧠 Key SQL Concepts Used

- JOINs (INNER JOIN, LEFT JOIN)
- GROUP BY & HAVING
- Aggregate functions (SUM, COUNT, AVG)
- Subqueries
- CTEs (Common Table Expressions)
- Data filtering and sorting
